# 三色成规

VonishOCR 的品牌色彩系统以三种核心色为锚点，辅以透明度阶梯与辅助色，构建出统一而克制的视觉秩序。

## 核心三色

### 掌灯青 #8FF6D2

掌灯青是 VonishOCR 唯一的彩色。它代表光、激活、当前状态、进度、复核定位。

使用场景：

- 当前选中的侧边栏项边框
- 识别中的扫描线
- 低置信字段的虚线框
- 按钮 hover 背景
- 成功状态图标
- 代码高亮关键字
- 链接文字与下划线

### 纸骨 #E7E1D0

纸骨色承载文本与信息。它是档案纸的颜色，是长时间阅读不疲劳的底色。

使用场景：

- 深色背景上的主文字
- 标题与正文
- 表单标签
- 表格表头文字
- 图标线条

### 炭黑 #11110F

炭黑是边界与背景。它不是纯黑，而是带着极微弱暖调的深炭色，避免纯黑的生硬与 OLED 屏幕的纯黑切割感。

使用场景：

- 主背景色
- 代码块背景
- 侧边栏背景
- 弹窗背景
- 状态条背景

## 透明度阶梯

### 掌灯青透明度

| 透明度 | 用途 |
|--------|------|
| 100% | 扫描线、当前指示器、链接文字 |
| 72% | 按钮 hover、激活状态填充 |
| 48% | 低置信字段边框、次要高亮 |
| 32% | 进度条背景、分隔线发光 |
| 18% | 按钮正常态背景 |
| 16% | 卡片 hover 背景 |
| 10% | 表格 hover 背景 |
| 8% | 极弱发光、状态提示背景 |

### 纸骨透明度

| 透明度 | 用途 |
|--------|------|
| 100% | 主文字、标题 |
| 64% | 正文、描述文字 |
| 32% | 占位符、禁用状态 |
| 24% | 分隔线、边框 |
| 16% | 背景纹理、水印 |
| 8% | 极淡背景、网格线 |

## 辅助色

辅助色不用于品牌表达，仅用于功能区分与状态提示。

| 颜色 | 色值 | 用途 |
|------|------|------|
| 钢灰 | #A39B8F | 次要文字、描述、时间戳 |
| 暗灰 | #1A1916 | 次级背景、卡片底色、代码块背景 |
| 面板灰 | #222026 | 弹窗背景、抽屉背景、控制台背景 |

## 状态色

| 状态 | 色值 | 用途 |
|------|------|------|
| 成功 | #8FF6D2（掌灯青）| 成功提示、完成标记 |
| 警告 | #D7B95A | 警告 Toast、异常标记、温度偏高 |
| 危险 | #B85A50 | 错误提示、删除确认、严重故障 |
| 信息 | #A39B8F（钢灰）| 普通提示、说明文字 |

## 禁止

以下用色方式在 VonishOCR 中被明确禁止：

- 渐变色（无论线性还是径向）
- 阴影（使用边框与透明度代替）
- 高饱和彩色（除掌灯青外，界面中不应出现其他纯色）
- 白色背景（浅色模式使用纸骨 `#E7E1D0` 作为最浅底色）

---

> 三色成规的约束看似严苛，实则是为了降低决策成本。当每一位设计师与开发者都无需争论「这个按钮用什么蓝」时，产品的一致性就得到了制度性保障。
