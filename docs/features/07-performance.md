# 性能模式

VonishOCR 提供三档性能模式，适配不同的算力环境、任务紧急程度与噪声容忍度。

## 野兽模式

| 参数 | 值 |
|------|-----|
| PL1 功耗限制 | 65W |
| 并发数 | 8 |
| 风扇策略 | 高性能 |
| 适用场景 | 紧急批量处理、服务器环境 |

野兽模式解锁 CPU/GPU 的完整性能。在配备独立显卡的台式机上，200 张发票的批量识别可压缩至 30 秒内完成。代价是风扇噪音显著增加、整机功耗上升、电池设备续航锐减。

建议在以下场景使用：

-  deadline 紧迫的批量发票识别
- 服务器机房环境，噪声不敏感
- 连接电源的笔记本，追求最短处理时间

## 均衡模式（默认）

| 参数 | 值 |
|------|-----|
| PL1 功耗限制 | 45W |
| 并发数 | 4 |
| 风扇策略 | 自动调速 |
| 适用场景 | 日常办公、文档处理 |

均衡模式是 VonishOCR 的默认设置。它在速度与舒适度之间取得平衡：风扇在轻负载时几乎无声，持续高负载时自动提速但不过激。日常识别单张图片时，用户几乎感知不到性能模式的存在。

## 节能模式

| 参数 | 值 |
|------|-----|
| PL1 功耗限制 | 25W |
| 并发数 | 2 |
| 风扇策略 | 静音优先 |
| 适用场景 | 老旧设备、后台处理、电池供电 |

节能模式将功耗与并发数压缩至最低，适合以下场景：

- 十年以上的老旧笔记本，散热能力有限
- 希望 VonishOCR 在后台静默完成批量任务
- 电池供电的离电场景，追求最长续航
- 夜间运行，风扇噪音必须最小化

## 切换方式

### 方式一：顶部状态条

1. 点击顶部状态条的当前性能模式文字（如「均衡模式」）
2. 弹出三档选择器，当前档位以掌灯青色高亮
3. 点击目标档位，切换即时生效

### 方式二：后端控制台

1. 点击顶部状态条「后端控制台」按钮
2. 在算力仪表盘下方找到「性能模式」区域
3. 通过下拉菜单切换，可查看各档位的详细参数

### 方式三：配置文件

直接编辑 `config.json`：

```json
{
  "performance": {
    "mode": "balanced",
    "pl1_limit": 45,
    "concurrency": 4
  }
}
```

修改后重启软件生效。

## 动态调度建议

对于混合负载的用户，建议采用「默认均衡 + 临时野兽」的策略：

- 日常单张识别：均衡模式，无感知运行
- 批量发票处理：切换野兽模式，完成后自动回退均衡（可在设置中开启「批量完成后自动回退」）
- 夜间归档任务：切换节能模式，放入后台，早晨收取结果

---

> 功耗控制依赖系统级接口（Windows Power API / macOS IOKit / Linux cpufreq），在虚拟机或容器环境中可能无法精确生效。此类环境下建议手动限制并发数而非依赖自动功耗控制。
