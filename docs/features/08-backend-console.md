# 后端控制台

后端控制台是 VonishOCR 的运维仪表板，提供算力监控、模型管理、API 配置与日志审计的一站式入口。

## 进入方式

点击顶部状态条右侧的「后端控制台」按钮，或按快捷键 `Ctrl + Shift + C`（macOS 为 `Cmd + Shift + C`）。

控制台以 Modal 弹窗形式覆盖主界面，底色为面板灰 `#222026`，1px 边框，无 drop-shadow，内发光营造机柜面板的质感。

## 算力仪表盘

仪表盘以卡片网格形式展示实时硬件状态：

| 指标 | 数据源 | 刷新频率 |
|------|--------|---------|
| GPU 显存占用 | nvidia-ml-py / DirectML API | 1s |
| GPU 利用率 | 同上 | 1s |
| GPU 温度 | 同上 | 1s |
| GPU 功耗 | 同上 | 1s |
| CPU 占用 | psutil | 1s |
| 内存占用 | psutil | 1s |
| 磁盘 I/O | psutil | 2s |
| 风扇转速 | 系统接口（若可用）| 2s |

仪表盘采用 JetBrains Mono 字体显示所有数值，确保等宽对齐。数值异常时（如温度 > 85°C），对应卡片边框变为警告黄色，数值变为危险红色。

## 模型驻留与切换

控制台中部展示当前模型状态：

- **已驻留模型**：显示当前加载的模型 ID、版本、驻留时长、显存占用
- **可切换模型**：列出已下载的三档模型，未下载的显示「需下载」
- **切换操作**：点击目标模型 → 确认卸载当前 → 加载目标 → 进度条 → 完成

模型切换期间，识别服务暂停，队列中的任务等待切换完成后继续执行。

## 性能模式切换

控制台提供比顶部状态条更详细的性能模式控制：

- 当前模式显示（含 PL1、并发数、风扇策略）
- 三档切换下拉菜单
- 自定义模式滑块（PL1 20W-80W，并发 1-16）
- 「批量完成后自动回退」开关

## 本地 API 管理

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 监听地址 | `127.0.0.1` | 仅本地访问 |
| 端口 | `8000` | 可修改为 1024-65535 |
| 服务状态 | 运行中 / 已停止 | 实时显示 |

操作按钮：

- **重启服务**：修改端口后必须重启
- **测试请求**：发送一张内置测试图到当前端点，验证服务可用
- **最近请求记录**：显示最近 20 次请求的耗时、状态码、端点

## 实时日志流

控制台底部为终端风格的实时日志流：

- 字体：JetBrains Mono，11pt
- 背景：炭黑 `#11110F`
- 时间戳：灰色，格式 `HH:MM:SS`
- 级别标签：`DEBUG` / `INFO` / `WARN` / `ERROR`，对应灰/青/黄/红
- 自动滚动：新日志到达时自动滚动到底部
- 筛选：按级别过滤、按标签过滤、按关键字搜索
- 导出：点击「导出 CSV」按钮，生成包含当前筛选结果的日志文件

## 队列监控

日志流上方显示当前队列摘要：

```
QUEUE: 0 | PROC: 2 | DONE: 67 | BATCH: 80 | ETA: 4m 32s
```

点击摘要展开详细队列面板，与主界面的队列面板数据同源。

---

> 后端控制台的设计灵感来自机柜面板与工业控制系统——信息密度高、反馈即时、操作明确。所有数值读数使用等宽字体，所有状态变化有颜色编码，所有危险操作需要确认。
