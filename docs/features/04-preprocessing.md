# 自适应预处理引擎

预处理是 OCR 精度的前置保障。VonishOCR 的自适应预处理引擎包含 5 步管线，每一步根据图像特征自动判断是否触发。

## 五步管线总览

```
原始输入 → [自动旋转] → [透视矫正] → [去横线+去噪+增强] → [质量评估] → 优化结果
```

每一步的触发条件与处理逻辑如下。

## 第一步：自动旋转

**触发条件**：检测到文本方向非水平（基于笔画方向直方图）  
**处理方式**：

- 检测主文本方向（0° / 90° / 180° / 270°）
- 对 90°/270° 场景（竖排文字）标记为「复杂版面」提示
- 对 180° 场景（倒置图片）执行旋正

旋转后的图像进入下一步处理。若旋转后文本方向仍不正常（如斜拍照片），触发透视矫正。

## 第二步：透视矫正

**触发条件**：检测到外边框倾斜角 > 5°，或四边形顶点偏离矩形 > 10%  
**处理方式**：

1. 检测文档轮廓（Canny 边缘 + 轮廓逼近）
2. 识别四边形顶点（若检测到完整边框）
3. 计算透视变换矩阵（getPerspectiveTransform）
4. 应用 warpPerspective 得到俯视图

对于无边框的文档（如零散纸张），透视矫正常常无法触发。此时依赖第三步的去斜处理。

## 第三步：去横线、去噪、增强

这一步包含三个并行子处理，各自独立触发。

### 去横线

**触发条件**：检测到水平线密度超过阈值（通常是有格线的纸张、试卷、表格）  
**处理方式**：

- Hough 变换检测水平线
- 判断线条是否为表格/格线（长度、间距、连续性）
- 使用 inpainting 算法去除线条，保留交叉文字

去横线对试卷处理至关重要。算法会保留答题框的边框线，仅去除引导横线。

### 去噪

**触发条件**：噪声频谱能量超过阈值  
**处理方式**：

- 高斯模糊去除高频噪声（适用于扫描噪声）
- 中值滤波去除椒盐噪声（适用于拍照噪点）
- 频域滤波去除周期性 Moire 纹（适用于低质量扫描）

去噪强度与场景分类联动：「退化图像」类应用最强去噪，「印刷清晰」类仅轻量去噪。

### 增强

**触发条件**：对比度 < 阈值 或 亮度不均  
**处理方式**：

- CLAHE（对比度受限自适应直方图均衡）：提升局部对比度
- 锐化卷积核：增强边缘清晰度
- 阴影去除：对光照不均的图片进行背景归一化

## 第四步：质量评估

预处理完成后，引擎对输出图像进行质量评分：

| 指标 | 阈值 | 低于阈值的处理 |
|------|------|-------------|
| 整体对比度 | 0.3 | 标记为退化图像，提示使用专业模型 |
| 边缘清晰度 | 0.4 | 加强锐化，二次增强 |
| 文本可读性 | 0.5 | 尝试超分辨率放大（2x）后重识别 |

质量评估的回退机制确保即使预处理无法挽救的极劣图像，系统也会给出明确的降级提示，而非返回无意义的乱码结果。

## 用户可见的预处理日志

识别中状态下，底部读数区滚动显示当前步骤：

```
自动旋转 → 正常 | 透视矫正 → 触发（倾斜角 12°）| 去横线 → 跳过 | 去噪 → 中值滤波 | 增强 → CLAHE | 质量评估 → 通过（0.72）
```

日志以微动画形式依次点亮，每个步骤名称在触发时以掌灯青色高亮 200ms，随后变为灰色完成态。

---

> 预处理引擎的设计原则是：能自动做的绝不打扰用户，但用户始终可以通过日志知道系统做了什么。当预处理结果不理想时，左侧工具抽屉提供手动覆盖选项，如「强制去横线」「跳过增强」等。
