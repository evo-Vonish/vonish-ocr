# 路线图

VonishOCR 的发展路线图按版本规划，从当前已实现的 v1.0 到规划中的 v2.0。

## v1.0：本地三档模型 + Diff 复核 + 批量处理

**状态**：已实现

- 本地三档 OCR 模型（极速/标准/专业）
- 场景自适应分类（8 类）
- 预处理流水线（10 步）
- Diff 级复核（原始/精修/Diff 三栏）
- 批量队列处理（200 张上限）
- 本地 API（localhost:8000）
- 后端控制台（算力仪表盘、日志审计）
- 导出功能（TXT/MD/DOCX/CSV）
- 系统托盘
- 四主题切换（dark/light/mono/hc）

## v1.1：AI Refiner 多厂商 + 流式输出

**状态**：已实现（核心功能），持续优化中

- AI 精修多厂商 failover（DeepSeek / OpenAI / Qwen）
- SSE 流式输出与打字机渲染
- AI Provider Center（方案管理、优先级、加密存储）
- 每日额度机制与自备 Key 中转
- 额度仪表盘与用量提醒

## v1.2：云端网页版 + 官方 API

**状态**：规划中

- 云端 GPU 加速网页版
- 官方 RESTful API（api.vonishocr.com）
- 账号系统与配置同步
- 云端批量处理（500 张上限）
- 开发者控制台（API Key 管理、用量审计）

## v1.3：后端控制台 + 主题系统完善

**状态**：开发中

- 后端控制台机柜面板视觉重构
- GPU 实时监控（NVIDIA / AMD / Intel）
- 实时日志 WebSocket / SSE 推送
- 性能模式动态调度（根据温度自动降频）
- 打印样式优化（纸面交还样张）

## v2.0：多模态

**状态**：远期规划

- 图像理解：不仅识别文字，还理解图片中的图表、示意图、流程图
- 表格结构识别：复杂表格的行列合并、嵌套表格
- 手写公式识别：将手写数学公式转换为 LaTeX
- 文档版面还原：保留原文档的多栏、页眉页脚、图文环绕布局
- 语义检索：对识别结果建立向量索引，支持自然语言搜索

## 版本发布节奏

| 版本 | 预计时间 | 发布类型 |
|------|---------|---------|
| v1.0.x | 2026 Q2 | 维护更新，bug 修复 |
| v1.1.x | 2026 Q3 | 功能更新，AI Refiner 完善 |
| v1.2 | 2026 Q4 | 大版本，云端版上线 |
| v1.3 | 2027 Q1 | 功能更新，控制台重构 |
| v2.0 | 2027 Q3 | 大版本，多模态能力 |

路线图会在 GitHub Discussions 的「Ideas」分类中持续更新，欢迎社区投票与讨论优先级的调整。

---

> 路线图不是承诺，而是方向。VonishOCR 的核心团队优先保证已发布版本的稳定性，新功能的开发节奏取决于社区需求与资源可用性。如果你对某个功能特别期待，请在 GitHub Discussions 中表达——用户的真实需求是路线图最好的指南针。
