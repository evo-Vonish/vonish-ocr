# 社区

VonishOCR 的社区通过 GitHub 托管，所有讨论、问题报告与功能请求都公开透明。

## GitHub Discussions

[GitHub Discussions](https://github.com/evo-Vonish/vonish-ocr/discussions) 是用户交流使用心得、分享技巧与提出问题的场所。

### 分类标签

| 分类 | 用途 |
|------|------|
| Q&A | 使用问题与解答 |
| Show and tell | 分享用 VonishOCR 完成的项目 |
| Ideas | 功能想法与初步讨论 |
| General | 其他话题 |

### 提问建议

1. 先搜索现有 Discussion，避免重复提问
2. 标题清晰概括问题（如「批量 200 张发票如何导出 CSV」）
3. 提供环境信息（操作系统、VonishOCR 版本、模型档位）
4. 如涉及识别质量问题，附上原图缩略图（注意脱敏）

## Issue 模板

提交 Issue 时请选择合适的模板：

| 模板 | 用途 | 标签 |
|------|------|------|
| Bug 报告 | 软件行为异常 | `bug` |
| 功能请求 | 希望新增的功能 | `feature` |
| 问题咨询 | 使用上的疑问 | `question` |
| 文档修正 | 文档错误或不清晰 | `docs` |

Issue 提交后，维护者会在 48 小时内回复分类。P0 级 bug（阻塞核心功能）会优先处理。

## 行为准则（Code of Conduct）

参与 VonishOCR 社区即表示同意以下行为准则：

### 我们的承诺

- 欢迎所有背景与经验的参与者
- 尊重不同的观点与经验
- 接受建设性批评
- 关注对社区最有利的事情
- 对其他社区成员表达共情

### 不可接受的行为

- 使用带有性暗示的语言或图像
- 挑衅、侮辱/贬损的评论，人身或政治攻击
- 公开或私下的骚扰
- 未经明确许可发布他人的私人信息（如物理地址、电子邮件）
- 其他在专业环境中被合理认为不适当的行为

### 执行

维护者有权利和责任删除、编辑或拒绝不符合本行为准则的评论、提交、代码、Issue 与其他贡献。维护者可暂时或永久性地禁止他们认为有不当、威胁、冒犯或有害行为的贡献者。

---

> 社区是产品的延伸。VonishOCR 的社区气质与产品气质一致：冷静、专业、尊重事实。这里没有「小白」与「大神」的等级，只有「提出问题的人」与「帮助解决问题的人」。
