# 竞品对比 · 主权之光

OCR 市场存在四类主流产品形态。以下是 VonishOCR 与这些类别的八维度对比。

|------|-----------|-------------|-------------|-------------|

## 类别说明

**传统在线 OCR**：将图片上传至服务商服务器，由服务端完成识别后返回结果。识别质量依赖网络延迟与厂商模型迭代，敏感文件存在合规风险。定价模式通常为订阅制或按调用次数计费。

**传统桌面 OCR**：在本地运行的 OCR 软件，识别过程不依赖网络。但多数产品模型单一、功能固化，缺乏场景自适应与 AI 后处理能力。部分产品的激活验证仍需联网。

**手机 OCR App**：面向移动端的照片识别工具，便利性高但功能浅层。预处理能力弱，批量处理困难，导出选项有限。大多数产品在免费使用数次后要求订阅。

## VonishOCR 的差异化定位

VonishOCR 的核心差异可以用三个词概括：**主权**、**精度**、**延展**。

**主权**：你的文件、你的模型、你的结果、你的配置，全部在本地。即使厂商停止服务，你已下载的模型仍可终身使用。

**精度**：三档模型自动调度 + 场景自适应预处理 + AI 精修后处理，构成一条完整的精度提升链路。从「能识别」到「识别对」，再到「识别好」，每一步都有工具支撑。

**延展**：本地 API 将 VonishOCR 从一款软件变成一个平台。任何能发送 HTTP 请求的语言或工具，都可以调用它的识别能力。Python 脚本、浏览器插件、Obsidian 插件、Alfred Workflow——接入方式只受想象力限制。
