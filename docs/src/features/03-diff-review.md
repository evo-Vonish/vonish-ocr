# Diff 级复核

Diff 级复核是 VonishOCR 的核心交互创新。它将原始 OCR、AI 精修与修改差异三栏并列，让用户对每一个被修改的字都有知情权与决定权。

## 三栏界面说明

识别完成后，右侧复核灯区域自动展开，默认显示「AI 精修」栏。三栏切换通过顶部的选项卡完成，切换动画时长 180ms。

### 原始 OCR 栏

显示模型直接输出的识别文本，不做任何后处理。这是「机器看到的内容」，用于判断模型本身的识别能力是否满足需求。若原始 OCR 已足够准确，可直接复制使用，无需触发 AI 精修。

### AI 精修栏

显示经大语言模型修正后的文本。修正范围包括：

- 错别字修正（如「仃」→「行」）
- 上下文补全（如被遮挡字符的合理推断）
- 格式规范化（如日期格式统一、标点补全）
- 专业术语校正（如医学名词、法律术语）

精修结果以流式方式渲染，SSE 连接建立后逐字追加，模拟打字机效果。

### Diff 栏

Diff 栏是复核的核心工具。它将原始 OCR 与 AI 精修进行字符级差异比对，以视觉化方式呈现所有修改。

## 错字修正标记

| 标记类型 | 视觉表现 | 含义 |
|---------|---------|------|
| 红色删除线 | ~~删除的文字~~ | AI 认为原始 OCR 错误，已移除 |
| 绿色新增 | <u>新增的文字</u> | AI 认为此处应补充或替换 |
| 黄色背景 | 高亮文字 | 上下文推断区域，原始 OCR 可能遗漏 |

Diff 列表的每行格式为：

```
原始片段 → 精修片段
```

示例：

```
夫君子之仃 → 夫君子之行
患者干 2026 → 患者于 2026
建议继续观案 → 建议继续观察
```

## 低置信字段标记

对于原始 OCR 中置信度低于阈值的字段（默认阈值 0.6），系统在三个栏中均以掌灯青色虚线框包围，并附带问号图标。

低置信字段的层级提示：

1. **外框**：掌灯青虚线矩形，标识字段边界
2. **角标**：右上角问号图标，悬停显示置信度数值
3. **标签**：Diff 栏底部显示「需复核」标签，汇总所有低置信字段数量

点击低置信字段，中央区域弹出原图局部放大切片，支持 2x/4x 缩放，方便人工核对。

## 复核操作

对 Diff 栏中的每一处修改，用户可执行以下操作：

| 操作 | 效果 | 快捷键 |
|------|------|--------|
| 确认 | 接受 AI 的修改，该标记变为灰色已确认状态 | Enter |
| 修改 | 点击文本进入编辑模式，手动输入正确字符 | 双击 |
| 忽略 | 拒绝 AI 的修改，恢复为原始 OCR 文本 | Delete |

复核状态会实时同步到 AI 精修栏，已确认的修改以绿色对勾标记，已忽略以灰色叉号标记。全部复核完成后，精修栏的文本即为最终输出。

---

> Diff 复核的设计理念是：AI 可以建议，但决定权在人。每一条红色删除线与绿色新增，都是一次机器与人类的对话。
