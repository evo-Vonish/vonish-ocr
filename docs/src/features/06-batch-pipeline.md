# 批量流水线

批量流水线是 VonishOCR 的并发任务调度系统，负责管理多图导入、队列排序、并发控制与失败重试。

## 队列面板

批量模式下，中央区域切换为队列面板，以卡片流展示每张图片的状态：

| 状态 | 颜色标识 | 说明 |
|------|---------|------|
| QUEUED | 纸骨灰 | 已导入，等待预处理 |
| PROC | 掌灯青（呼吸动画）| 正在识别或 AI 精修 |
| DONE | 暗灰（实心）| 已完成，结果已写入 |
| 异常 | 暗红 | 识别失败，已重试 1 次仍失败 |

队列面板顶部显示总进度：`已完成 / 处理中 / 等待中 / 排队中 / 总数`。如 `67/3/50/80/200` 表示 67 张完成、3 张处理中、50 张等待、80 张排队、共 200 张。

## 流程图

```
导入 → [自动场景分类] → [预处理] → [OCR 识别] → [AI 精修（可选）] → [结果写入] → 完成
```

每一步的实际执行由 `LocalQueue` 调度，并发数受性能模式控制：

| 性能模式 | 并发数 | 适用场景 |
|---------|--------|---------|
| 野兽模式 | 8 | 紧急批量，追求最短时间 |
| 均衡模式 | 4 | 日常使用，平衡速度与功耗（默认）|
| 节能模式 | 2 | 老旧设备，后台静默处理 |

## 失败重试

单图在流水线中任一环节失败，系统执行以下策略：

1. **自动重试 1 次**：同一图片重新放入队列末尾，使用相同配置重试
2. **仍失败则标记异常**：图片卡片变为暗红色，结果栏显示失败原因摘要
3. **异常汇总**：全部完成后，通知面板汇总所有异常图片的数量与原因

常见失败原因：

| 原因 | 占比 | 处理方式 |
|------|------|---------|
| 预处理异常 | 35% | 手动跳过增强或透视矫正 |
| 模型推理错误 | 25% | 切换模型档位后重试 |
| 显存不足 | 20% | 降低并发数或切换到节能模式 |
| AI 精修超时 | 15% | 检查网络或关闭 AI 精修 |
| 文件损坏 | 5% | 修复或替换源文件 |

---

> 批量任务支持运行时动态调整并发数：在队列面板中拖动并发滑块，后端会立即释放或增加工作线程，无需中断已开始的任务。
