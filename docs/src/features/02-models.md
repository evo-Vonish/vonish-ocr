# 三档模型

VonishOCR 内置三档 OCR 模型，覆盖从速度优先到精度优先的全谱需求。系统默认自动调度，也允许用户手动锁定。

## 极速模型：rapidocr-mobile-cn

| 属性 | 规格 |
|------|------|
| 体积 | 22 MB |
| 语言 | 中英双语 |
| 能力 | 文字识别 |
| 推理速度 | 单张 < 0.3s（GPU）/ < 1s（CPU）|
| 适用场景 | 打印文档、屏幕截图、高对比度发票 |

极速模型基于轻量级 CNN 架构，专为移动端与实时场景设计。在印刷清晰类文档上，其精度与标准模型差距在 2% 以内，但速度提升 3-5 倍。是批量处理首选模型。

## 标准模型：cnocr-standard-cn

| 属性 | 规格 |
|------|------|
| 体积 | 80 MB |
| 语言 | 中英双语 |
| 能力 | 文字识别 + 基础表格结构 |
| 推理速度 | 单张 1-2s（GPU）/ 3-5s（CPU）|
| 适用场景 | 通用场景、手写材料、混合版面 |

标准模型是 VonishOCR 的默认模型，在精度与速度之间取得均衡。它支持基础表格结构识别，可将简单表格输出为 Markdown 或 CSV 格式。对于绝大多数用户，标准模型是日常使用的最佳选择。

## 专业模型：paddleocr-vl-1.5

| 属性 | 规格 |
|------|------|
| 体积 | 1.87 GB |
| 语言 | 中英双语 |
| 能力 | 文字识别 + 表格结构 + 版面分析 + 公式检测 |
| 推理速度 | 单张 3-5s（GPU）/ 10-20s（CPU）|
| 适用场景 | 复杂版面、退化图像、古籍、试卷 |

专业模型是目前 VonishOCR 中能力最强的模型。它的版面分析模块可以识别多栏、图文混排、页眉页脚等复杂结构，并以结构化格式输出。公式检测模块支持将印刷公式转换为 LaTeX 表达式。

> 注意：当前版本的专业模型（PaddleOCR VL 1.5）使用 HuggingFace Transformers 格式（safetensors），VonishOCR 的 ONNX 引擎正在进行格式转换适配。在适配完成前，专业模型档位在 UI 中显示为「即将可用」状态。

## 自动匹配逻辑

系统根据以下信号自动选择模型：

```
if 场景分类 == "印刷清晰" 且 图片尺寸 < 2MP:
    推荐极速模型
elif 场景分类 == "退化图像" 或 "复杂版面":
    推荐专业模型（如已下载）
    未下载时 fallback 到标准模型
else:
    推荐标准模型
```

用户手动锁定模型后，自动匹配逻辑不再生效，所有后续图片均使用锁定模型处理。

## 模型缓存管理

模型加载后驻留显存/内存中，以避免重复加载的耗时。模型缓存管理策略：

| 操作 | 触发条件 | 效果 |
|------|---------|------|
| 自动加载 | 首次识别、切换档位 | 加载对应模型至显存 |
| 自动卸载 | 10 分钟无识别任务 | 释放显存，保留内存映射 |
| 手动切换 | 用户点击模型切换 | 卸载当前 → 释放显存 → 加载目标 |
| 清理缓存 | 后端控制台「清理缓存」按钮 | 卸载所有模型，释放全部显存 |

显存占用提示：

- 极速模型：约 150 MB 显存
- 标准模型：约 500 MB 显存
- 专业模型：约 3 GB 显存

当系统检测到显存不足时，会弹窗提示「显存紧张，建议降低并发数或切换至节能模式」。

---

> 模型文件存放于 `models/` 目录，命名格式为 `{model_id}_{version}.onnx`。模型元数据由 `manifest.json` 描述，包含版本、大小、适用场景与 SHA256 校验值。
