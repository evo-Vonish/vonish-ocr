# 主题系统

VonishOCR 提供四套主题，覆盖正常视觉、打印输出、无障碍访问与系统级兼容。

## 深色主题（默认）

| 元素 | 颜色 | 说明 |
|------|------|------|
| 桌面背景 | `#11110F` 炭黑 | 主工作区背景 |
| 文件/文本 | `#E7E1D0` 纸骨 | 主文字、图片边框 |
| 焦点/激活 | `#8FF6D2` 掌灯青 | 当前选中项、扫描线、进度条 |
| 次级背景 | `#1A1916` 暗灰 | 卡片、代码块、抽屉 |
| 边框 | `#3D383F` 钢灰 24% | 分隔线、面板边界 |

深色主题是 VonishOCR 的默认与推荐主题。它在暗光环境下减少眼部疲劳，同时让掌灯青的提示效果最为醒目。

## 黑白主题

黑白主题面向打印输出与单色显示器场景。

| 元素 | 颜色映射 |
|------|---------|
| 掌灯青 | 映射为白色 `#FFFFFF` |
| 纸骨 | 映射为白色 `#FFFFFF` |
| 炭黑 | 映射为黑色 `#000000` |

Diff 标记的彩色编码在黑白主题中改为纹理编码：

| 原始含义 | 黑白主题表现 |
|---------|-------------|
| 红色删除线 | 删除线（`text-decoration: line-through`）|
| 绿色新增 | 下划线（`text-decoration: underline`）|
| 黄色背景 | 方括号包裹 `[ ... ]` |
| 低置信虚线框 | 虚线边框（`border-style: dashed`）|

禁止在黑白主题中使用任何彩色。所有状态通过线型、纹理与灰度区分。

## 高对比度主题

高对比度主题面向视觉障碍用户与强光环境。

| 修改项 | 规则 |
|--------|------|
| 边框 | 统一 2px 实线，去除所有 1px 细边框 |
| 透明度 | 去除所有透明度，所有颜色使用 100% 不透明度 |
| 发光 | 去除内发光与外发光效果 |
| 低置信 | 三层可见：外框（2px 实线）+ 角标（方块）+ 「需复核」文字标签 |
| 焦点 | 所有可聚焦元素必须有 2px 掌灯青轮廓线 |

## 系统级兼容

VonishOCR 响应以下系统级媒体查询：

```css
@media (prefers-contrast: more) {
  /* 自动启用高对比度变体 */
}

@media (prefers-color-scheme: light) {
  /* 系统浅色模式时可选自动切换 */
}

@media (forced-colors: active) {
  /* Windows 高对比度模式时的强制适配 */
}
```

在 `forced-colors` 模式下，VonishOCR 使用系统强制配色表（ButtonText、Canvas、Highlight 等），确保与操作系统级的高对比度设置无缝配合。

## 主题切换方式

1. **状态条切换**：点击顶部状态条的主题图标，弹出四宫格选择器
2. **设置页切换**：配置抽屉 → 外观 → 主题
3. **快捷键**：`Ctrl + Shift + T`（macOS `Cmd + Shift + T`）循环切换

主题选择持久化至 `config.json`，重启后保持。

---

> 主题系统的设计原则是「不遗漏任何用户」。视力正常的开发者使用深色主题进行调试，视觉障碍用户在高对比度主题下获得完整的操作能力，档案员在黑白主题下直接打印界面截图作为纸质记录——四套主题覆盖全部场景，没有例外。
