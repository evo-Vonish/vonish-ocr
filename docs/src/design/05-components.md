# 组件规范

VonishOCR 的 UI 组件系统遵循工业面板风格：直角优先、边框定义边界、透明度传达层级、发光替代阴影。

## 弹窗 Modal

- **背景**：面板灰 `#222026`
- **边框**：1px 实线，钢灰 `#3D383F`
- **圆角**：4px（不大于 8px 的红线）
- **阴影**：禁止 drop-shadow，使用内发光替代（`box-shadow: inset 0 0 20px rgba(143, 246, 210, 0.04)`）
- **标题**：Noto Serif SC，18pt，600 字重
- **内容区**：Noto Sans SC，14pt，行高 1.6
- **按钮区**：右对齐，主按钮在左、次按钮在右
- **关闭方式**：右上角 × 图标、ESC 键、点击遮罩层
- **最大宽度**：560px，移动端全屏

## 轻量提示 Toast

- **位置**：底部居中，距底边 48px
- **背景**：暗灰 `#1A1916` + 1px 钢灰边框
- **圆角**：4px
- **状态色**：
  - 成功：左侧 3px 掌灯青边框
  - 警告：左侧 3px 警告黄边框
  - 错误：左侧 3px 危险红边框
  - 信息：左侧 3px 钢灰边框
- **进度条**：底部 2px 线条，3 秒内从 100% 宽度消退至 0%
- **动画**：从底部滑入（translateY 16px → 0），200ms ease-out

## 选项卡 Tabs

### 水平变体

- 用于复核灯区域的三栏切换
- 标签文字：Noto Sans SC，13pt
- 指示器：底部 2px 掌灯青线条
- 切换动画：指示器水平滑动，180ms ease-in-out
- 未选中标签文字：钢灰 `#A39B8F`
- 选中标签文字：纸骨 `#E7E1D0`

### 垂直变体

- 用于设置页的多级分类
- 指示器：左侧 2px 掌灯青线条
- 布局：左侧 200px 标签栏 + 右侧内容区

### 胶囊变体

- 用于性能模式、主题切换的二选一/三选一
- 背景：暗灰 `#1A1916`
- 选中项：掌灯青背景 + 炭黑文字
- 未选中项：透明背景 + 钢灰文字
- 圆角：最大 8px（胶囊两端）

## 确认对话框 Confirm

- **常规操作**：直接确认，无二次验证
- **危险操作**（删除模型/清除缓存/删除文件）：需要输入确认文字解锁
  - 提示文字：「输入「删除」以确认操作」
  - 输入框：等宽字体，危险操作输入正确后确认按钮从禁用变为可用
  - 目的：防止习惯性点击造成的误操作

## 底部抽屉 Drawer

- **专用场景**：移动端（<768px）的工具抽屉与复核灯
- **触发**：点击底部导航图标或从屏幕边缘侧滑
- **关闭**：点击遮罩、手势下滑、点击关闭按钮
- **动画**：从底部滑入，300ms ease-out
- **高度**：最大占屏幕高度的 85%，内容超出可滚动

## 加载 Loading

VonishOCR 明确禁止旋转圆圈（spinner）作为加载指示器。允许的加载形式：

### 扫描线动画

- 一条掌灯青色的水平线，从图片顶部移动到底部
- 速度：1-2 秒一个周期
- 寓意：模拟实体扫描仪的工作过程

### 管线节点依次点亮

- 预处理管线的 5 个步骤以节点形式排列
- 当前步骤的节点以掌灯青点亮，已完成步骤以纸骨常亮，未开始步骤以钢灰暗亮
- 节点间以 1px 线条连接，线条随进度填充掌灯青

### 进度条

- 批量任务使用水平进度条
- 高度 4px，掌灯青填充，暗灰背景
- 顶部显示「已完成 / 总数」文字

---

> 组件规范的终极目标是「可预测性」。当用户第二次使用 VonishOCR 时，他应该能凭直觉判断每个元素的用途与交互方式。一致性不是美学追求，而是认知效率的保障。
