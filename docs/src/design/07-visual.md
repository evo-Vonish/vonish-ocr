# 海报与视觉

VonishOCR 的品牌视觉体系包含海报系列、Logo 规范与应用图标标准。

## 品牌海报系列

### 海报 1：本地部署优势

- **主题**：文件不离开设备，算力主权在你手中
- **视觉**：档案室的冷光台灯照亮一张正在识别的老旧照片，光线边缘有微弱的掌灯青色
- **文案**：「光在手里」
- **规格**：A3 纵向，300dpi，CMYK

### 海报 2：完整生态

- **主题**：本地版 / 云端版 / API 三形态覆盖全部场景
- **视觉**：三台设备（笔记本、浏览器窗口、服务器机柜）以等距视角排列，每台设备的屏幕发出相同的掌灯青光
- **文案**：「算力在你这，也在你那边」
- **规格**：A3 纵向，300dpi，CMYK

### 海报 3：竞品对比

- **主题**：八维度对比，主权之光
- **视觉**：极简表格设计，VonishOCR 列以掌灯青高亮，其余列以纸骨灰度呈现
- **文案**：「你的文件，不需要第三只眼」
- **规格**：A3 纵向，300dpi，CMYK

### 海报 4：API 接入指南

- **主题**：一行代码，本地 OCR
- **视觉**：代码编辑器截图风格，掌灯青色语法高亮，背景为炭黑
- **文案**：`requests.post('http://localhost:8000/v1/ocr')`
- **规格**：A3 纵向，300dpi，CMYK

## 海报设计规范

| 项目 | 规范 |
|------|------|
| 底色 | 炭黑 `#11110F` 或暗灰 `#1A1916` |
| 主文字 | 纸骨 `#E7E1D0`，Noto Serif SC |
| 副文字 | 钢灰 `#A39B8F`，Noto Sans SC |
| 强调 | 掌灯青 `#8FF6D2`，用于标题关键字、数据高亮、边界线 |
| 数据/代码 | JetBrains Mono，等宽 |
| 边距 | 15mm |
| 出血 | 3mm |
| 分层 | PSD 分层可编辑，文字层保留矢量 |

## Logo 规范

### 概念

VonishOCR 的 Logo 概念为「深海仪器」与「纸张扫描光束」的结合：

- 深海仪器：精密、工业、无装饰、功能性
- 纸张扫描光束：一道掌灯青色的光从上到下扫描过一张纸的轮廓

### 配色

| 版本 | 用途 |
|------|------|
| 主色版 | 掌灯青光束 + 纸骨纸张 + 炭黑背景 |
| 反色版 | 炭黑光束轮廓 + 纸骨纸张 + 掌灯青背景 |
| 单色版 | 纯纸骨线条（用于水印、底纹） |

### 安全空间

Logo 周围必须保留等于 Logo 高度 20% 的空白区域，禁止任何文字或图形侵入。

### 最小尺寸

- 数字屏幕：24px 高度
- 印刷品：10mm 高度

低于最小尺寸时，使用简化的「V」字形图标替代完整 Logo。

## 应用图标

| 平台 | 规格 | 风格 |
|------|------|------|
| Windows | .ico，256x256 主尺寸 | 炭黑背景，纸骨纸张，掌灯青光束 |
| macOS | .icns，1024x1024 主尺寸 | 同上，遵循 macOS Big Sur 圆角规范 |
| Linux | .png，512x512 | 同上，直角，无平台特定风格 |
| Android | 自适应图标，前景+背景层 | 前景为 Logo，背景为炭黑 |

---

> 视觉系统的输出物（海报、Logo、图标）必须与界面系统保持一致的品牌气质。如果一个海报看起来像是 SaaS 公司的营销物料，那它就不属于 VonishOCR。
