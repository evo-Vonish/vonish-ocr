# 场景指南

VonishOCR 的场景自适应引擎能够自动判断输入图像的类型，并匹配最佳的预处理流程与模型档位。以下是五种高频典型场景的操作指南。

## 发票批量识别

**输入特征**：印刷清晰、排版规整、字段位置固定  
**推荐模型**：极速模型（rapidocr-mobile-cn，22MB）  
**典型耗时**：单张 < 0.5s，200 张批量约 90s（均衡模式）

操作流程：

1. 将发票扫描件或拍照件拖入中央拖拽区
2. 系统自动检测为「印刷清晰」类，加载极速模型
3. 如超过 20 张，自动进入批量流水线模式
4. 识别完成后，选择「导出 → 结构化表格」，生成 Excel 或 CSV

批量模式下建议开启野兽模式以缩短总耗时。若发票存在折痕或阴影，预处理引擎会自动触发透视矫正与对比度增强。

## 古籍数字化

**输入特征**：竖排文字、复杂版面、低对比度、纸张老化  
**推荐模型**：专业模型（paddleocr-vl-1.5，1.87GB）  
**典型耗时**：单张 3-5s

操作流程：

1. 拖入古籍扫描页，系统可能分类为「复杂版面」或「退化图像」
2. 如自动分类不准确，点击左侧场景标签栏手动指定「复杂版面」
3. 确认加载专业模型（如未下载，需先放置模型文件）
4. 识别完成后，进入 Diff 复核页，逐字比对红色删除线与绿色新增
5. 对低置信字段（掌灯青虚线框），点击展开原图切片确认

竖排识别建议手动覆盖场景分类，因为自动分类器可能将竖排古籍误判为「低质量扫描」。

## 试卷自动批改

**输入特征**：手写填涂 + 印刷横线、去横线需求、公式与符号  
**推荐模型**：标准模型（cnocr-standard-cn，80MB）  
**典型耗时**：单张 1-2s

操作流程：

1. 扫描或拍摄试卷，确保光线均匀、无强烈阴影
2. 系统检测为「手写材料」或「试卷」类，自动触发去横线预处理
3. 手写体识别完成后，对公式区域可手动触发 AI 精修
4. AI Refiner 会尝试将 `sqrt(x)` 等手写形式转换为标准 LaTeX
5. 导出为 Markdown 或 DOCX，便于导入教务系统

去横线预处理在检测到水平线密度超过阈值时自动触发。若试卷有彩色答题框，预处理引擎会保留框线仅去除横格线。

## 手写笔记整理

**输入特征**：任意角度手写文字、多页笔记、归档需求  
**推荐模型**：标准模型（cnocr-standard-cn）  
**典型耗时**：单张 1-2s

操作流程：

1. 批量拍摄或扫描手写笔记
2. 系统自动旋转矫正（检测文本方向，支持 90°/180°/270° 翻转）
3. 识别结果按页码顺序进入批量队列
4. 完成后合并导出为单个 Markdown 文档，或 ZIP 分包保存

对于潦草字迹，建议识别后开启 AI 精修进行上下文补全。对于专业术语密集的领域笔记（如医学、法律），建议在 AI Provider Center 中选择针对性更强的厂商方案。

## 屏幕截图取字

**输入特征**：屏幕截图、UI 文字、高对比度、边缘锐利  
**推荐模型**：极速模型（rapidocr-mobile-cn）  
**典型耗时**：单张 < 0.3s

操作流程：

1. 直接拖入截图或粘贴剪贴板图像
2. 系统瞬间识别为「印刷清晰」类，秒级返回结果
3. 点击复制按钮即可获取纯文本
4. 若截图包含代码，AI 精修可尝试格式化缩进与语法修复

截图取字是 VonishOCR 最快的使用路径，从粘贴到复制通常不超过 3 秒。

---

> 场景分类器基于传统 CV 规则（<5ms），结合图像特征、文本密度与笔画分析进行判断。当自动分类与预期不符时，随时可通过左侧工具抽屉手动覆盖。
