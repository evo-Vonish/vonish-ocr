# 首次识别

安装完成并确认后端就绪后，按照以下步骤在 3 分钟内完成第一次 OCR 识别。

## 第一步：打开软件

启动 VonishOCR 后，你会看到主界面的「空状态」：

- 中央区域为拖拽区，显示「拖入图片或点击选择」提示
- 左侧为工具抽屉（280px），包含场景分类、模型切换与文件列表
- 右侧为复核灯区域（320px），当前处于收起状态
- 顶部状态条 48px，显示后端状态、模型档位与性能模式
- 底部读数 40px，显示当前任务状态

## 第二步：拖入图片

从文件管理器中拖入一张 JPG 或 PNG 图片到中央拖拽区。支持的格式包括：

- `JPEG` / `JPG`
- `PNG`
- `WebP`
- `BMP`（自动转换）
- `TIFF`（自动转换）

拖入后，图片缩略图出现在左侧文件列表中，中央区域进入「识别中」状态。

## 第三步：观察识别过程

识别中界面会依次显示：

1. **扫描线动画**：从图片顶部到底部的绿色扫描线，模拟文本扫描过程
2. **预处理日志**：在底部读数区显示当前预处理步骤（如「自动旋转 → 去斜 → 降噪」）
3. **模型切换提示**：如系统判断需要更高精度模型，会显示「正在加载专业模型」进度条
4. **结果浮现**：识别完成后，扫描线消失，右侧复核灯区域自动展开

## 第四步：查看三栏结果

右侧复核灯区域展示三栏内容：

| 标签 | 内容 | 用途 |
|------|------|------|
| 原始 OCR | 模型直接输出的识别文本 | 查看原始识别质量 |
| AI 精修 | 经大语言模型修正后的文本 | 获取最优可读性结果 |
| Diff | 原始与精修的逐字差异 | 复核哪些字被修改过 |

Diff 栏用红色删除线标记被删除的字，绿色新增标记替换后的字。格式示例：

```
夫君子之仃 → 夫君子之行
```

## 第五步：关注低置信标记

若识别结果中存在不确定字符，这些字段会被掌灯青色的虚线框包围，并附带问号图标。悬停时显示提示：「置信度 0.43，建议复核」。

点击低置信字段，中央区域会弹出该字段在原图中的局部放大切片，方便对照确认。

## 空状态界面速查

| 状态 | 界面表现 | 含义 |
|------|---------|------|
| 空状态 | 中央显示拖入提示，扫描线静止 | 等待输入 |
| 识别中 | 扫描线动画，底部显示预处理步骤 | 正在处理 |
| 结果展示 | 右侧复核灯展开，三栏可切换 | 识别完成 |
| 低置信 | 文本中穿插虚线框与问号 | 需人工复核 |

---

> 首次识别建议使用打印清晰的文档或截图，以获得最佳体验。手写材料、古籍等复杂场景请参阅[场景指南](/product/03-scenarios)。
